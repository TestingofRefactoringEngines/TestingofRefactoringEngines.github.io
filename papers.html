Chatgpt-Based Test Generation for Refactoring Engines Enhanced by Feature Analysis on Examples

Abstract: We propose a ChatGPT-based approach to testing refactoring engines. We first manually analyze bug reports and test cases associated with refactoring engines, and construct a feature library containing fine-grained features that may trigger defects in refactoring engines. The approach automatically generates prompts according to both predefined prompt templates and features randomly selected from the feature library, requesting ChatGPT to generate test programs with the requested features. Evaluation on four engines suggests that the proposed approach identified a total of 115 previously unknown bugs.

Cross-Refactoring-Type Test Program Migration for Refactoring Engines

Abstract: Refactoring engines suffer from software bugs, and buggy engines may silently inject fatal errors into projects. This paper proposes an automated approach called EngineTest to test refactoring engines. It leverages LLMs to migrate known bug reports from one refactoring type to others, identifying 85 unknown bugs across Java, C/C++, and Python.

Testing Method Relocation Algorithms via Template-Based Systematic Structural Traversal and Precondition Filtering

Abstract: We propose RelocTest, a comprehensive testing framework that combines template-driven structural traversal with automated precondition filtering. It systematically explores input space by populating skeletons with LLM-guided completion. Evaluation on 7 mainstream engines uncovered 56 unknown bugs.
