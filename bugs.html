Compilation Error

Eclipse JDT Case #1868

Before: Pull Up m() from C
class B { }
class A extends B {
  static public void m() { }
}
class C extends B {
  void m() { }
}
After: Illegal Shadowing in A
class B {
  void m() { }
}
class A extends B {
  static public void m() { }
  // Error: static m() cannot hide instance m()
}
Analysis: The engine moved C.m() to parent B but failed to check siblings. In A, a static method cannot hide an inherited instance method from B, causing a compilation error.
Semantic Drift

Eclipse JDT Case #1750

Before: Rename foo2 to foo1
static void foo1(Number n) { out("1"); }
static void foo2(Long i) { out("2"); }
public static void main(String[] args) {
  long n = 0; foo1(n);
}
After: Overload Resolution Changed
static void foo1(Number n) { out("1"); }
static void foo1(Long i) { out("2"); }
public static void main(String[] args) {
  long n = 0; foo1(n);
  // Binds to foo1(Long), output changes to "2"
}
Analysis: Renaming foo2 to foo1 introduced a more specific overload for the call in main(). The program behavior silently changed, leading to semantic drift.
Compilation Error

Refactoring Case #2314

Before: Inline method()
static class Condition1 {
  static int method() { return 42; }
}
VoidFunction v = Condition1::method;
After: Broken Method Reference
static class Condition1 { }
VoidFunction v = Condition1::method;
// Error: method() undefined in Condition1
Analysis: The engine failed to update Java 8+ method references. Deleting the method without checking its reference usage breaks functional interface bindings.
Compilation Error

Refactoring Case #2307

Before: Move m() to TargetClass
void m(TargetClass<String> target) {
  String v = target.genericField;
}
After: Generic Type Mismatch
class TargetClass<T> {
  void m() {
    String v = genericField;
    // Error: cannot convert from T to String
  }
}
Analysis: Moving a method into a generic class caused the loss of concrete type information. The field access changed from String to T, breaking type safety.
Semantic Drift

Refactoring Case #1757

Before: Extract to m()
class B extends A {
  void f() { m("1"); } // Calls A.m
}
After: Accidental Hiding
class B extends A {
  void f() { m("1"); } // Now calls B.m!
  private static void m(String s) { }
}
Analysis: Extracting a method with a name that already exists in the parent class redirect existing calls, changing the program logic.
